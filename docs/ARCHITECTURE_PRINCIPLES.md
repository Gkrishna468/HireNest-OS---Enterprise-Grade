# Architecture Principles

1. **Understand First**: Never modify code before understanding the surrounding architecture.
2. **Inspect Existing**: Always inspect existing services before creating new ones.
3. **DRY & Reuse**: Reuse existing repositories, services, events, hooks, and components. Never duplicate functionality.
4. **Backward Compatibility**: Never break backward compatibility.
5. **Quality Standards**: Every implementation must satisfy SOLID, Clean Architecture, Repository Pattern, Dependency Injection, Type safety, Production logging, and Error handling.
6. **Mindset**: Think like a Principal Engineer at Google.
