# Implementation Proposal Template

Every feature proposal MUST contain the following 11 sections. Code generation cannot begin until this template is populated and explicitly approved.

1. **Existing implementation found**
2. **Files inspected** (with strict citations)
3. **Components reused**
4. **Architecture impact**
5. **Firestore impact** (Estimated Reads/Writes)
6. **Event Bus impact**
7. **AI Gateway impact** (Estimated AI Cost)
8. **Performance impact** (API Latency & Cloud Cost)
9. **Security impact** (Operational risk & ABAC changes)
10. **Testing plan**
11. **Rollback plan**
